# Security Policy

## Supported Versions

We provide security updates for the following versions of pastebinit:

| Version | Supported          |
| ------- | ------------------ |
| 1.x     | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

We take security issues seriously and appreciate you reporting them responsibly.

### How to Report

Send an email to **security@example.com** (replace with your actual security email) with the following information:

- A detailed description of the vulnerability
- Steps to reproduce the issue
- Which version of pastebinit is affected
- Any proof-of-concept code (if applicable)

### What to Expect

- **Acknowledgment:** You will receive an acknowledgment within 48 hours
- **Status Updates:** We will update you on the status of your report at least every 5 business days
- **Timeline:** We aim to resolve critical vulnerabilities within 30 days

### Process

1. The vulnerability is reported and acknowledged
2. We investigate and assess the scope
3. A fix is developed and tested
4. A security update is released
5. Public disclosure is coordinated with the reporter

### What Happens Next?

- **Accepted vulnerability:** We work on a fix and keep you informed of the progress
- **Declined report:** We explain why and provide feedback

We follow the principle of responsible disclosure and ask that you do not publicly disclose the vulnerability until we have had reasonable time to address it.

Thank you for helping make pastebinit more secure!
